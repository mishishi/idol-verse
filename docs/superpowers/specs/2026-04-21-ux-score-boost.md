# UX 分数从 6 提升到 7.5 — 设计规范

**日期**: 2026-04-21
**目标**: 提升体验质量，建立偶像感设计语言
**整体方向**: 从"功能可用"走向"让人想打开 app"

---

## 1. 首页信息层次重构（视觉优先级）

### 问题
8 个内容区块平铺，每个都有视觉权重，用户不知道该先看哪里。

### 方案：重新定义"第一眼"
首页按注意力流设计：

```
[Hero Banner — 活动通知，仅在最顶部]
     ↓
[Currency Strip — 资源位，左上角，小而轻]
     ↓
[Stats Center — 数据骄傲，放大突出]
     ↓
[Quick Actions — 2x2 操作入口，最核心的两个]
     ↓
[Scroll Sections — 次要内容，可视情况浏览]
     ↓
[Bottom Quickbar — 导航锚点，固定底部]
```

### 具体改动
1. **Hero Banner** 保持，但降低视觉权重（去掉 glow，只保留文字引导）
2. **Currency Strip** 缩小字号，从 `1rem` → `0.75rem`，从 `home-hero-banner` 内移到 `header` 旁
3. **Quick Actions** 从 6 卡片 → 4 卡片（召唤 + 应援 + 图鉴 + 排行），2 列网格
4. **Scroll Sections** 从 4 卡片 → 3 卡片，移除公告栏
5. **移除** Portal Background 装饰圈（干扰主内容）

## 2. 抽卡体验戏剧化

### 问题
抽卡结果就是一张卡片翻过来，缺少心跳时刻。

### 方案：三段式节奏

```
阶段 1：等待（1.5s）
  → 全屏暗化，中央显示"召唤中..."文字，粒子聚拢效果
  → 声音提示（可选）

阶段 2：揭示（0.8s）
  → 卡片从黑暗中弹出，带有 SSR/UR 的发光边框
  → 高稀有度卡片有更强的光效

阶段 3：确认（停留）
  → 卡片持续发光，用户确认后关闭
  → 显示"恭喜获得！"文字
```

### 具体改动
- `GachaPage` 的 `pulling` 状态增加全屏遮罩
- 结果卡片根据稀有度显示不同强度的 `neon glow`
- UR 结果增加背景金色粒子爆炸效果
- 抽卡按钮在 pull 中时禁用并显示 loading

## 3. 节奏游戏偶像化

### 问题
三轨道tap/hold界面和偶像主题完全割裂。

### 方案：融入 Idol 元素

```
轨道设计：
  → 三轨道改为三位偶像角色立绘
  → 音符从角色方向飞来
  → Perfect/Great/Miss 文字以应援棒荧光棒样式显示

判定反馈：
  → Perfect: 偶像立绘发光 + 粒子爆发
  → Miss: 偶像立绘轻微暗淡
  → Combo 越高，背景特效越华丽
```

### 具体改动
- `useRhythmGame.ts` 导出 `combo` 状态
- 音符击中时触发偶像表情变化（通过 CSS class 切换）
- 背景根据 combo 等级切换不同滤镜

## 4. 偶像感动效体系

### 问题
动效缺乏目的性，没有"偶像感"。

### 方案：建立动效语义

| 动效类型 | 使用场景 | 风格 |
|----------|----------|------|
| **呼吸感** | 首页背景星云、应援殿偶像立绘 | 缓慢、温柔、周期长（8-12s） |
| **心跳感** | 抽卡结果、SSR/UR 获得时 | 快速放大 + 轻微回弹 |
| **闪耀感** | 召唤按钮、偶像获得金色边框 | 间歇性光点闪烁 |
| **流畅感** | 页面切换、模态框打开 | 0.3s ease-out，无弹性 |

### 移除项
- 所有 hover 时的 `translateY` 上浮（已做）
- 星星闪烁（Gallery 以外的页面）
- 粒子漂浮（非抽卡场景）

## 5. 移动端适配

### 问题
完全没有响应式，tablet/手机体验差。

### 方案：渐进增强

```
桌面（>1024px）: 完整 2-3 列布局
平板（768-1024px）: 2 列网格，底部导航保持
手机（<768px）: 单列，底部 Tab 导航，Hero Banner 简化
```

### 具体改动
- `home.css` 的 `.actions-grid` 改为响应式：`repeat(2, 1fr)` → `repeat(2, 1fr)` on tablet, `1fr` on mobile
- 底部 `Quickbar` 在手机端改为 `BottomNav`
- Stats Center 在手机上改为横向滚动

## 6. 字体与留白

### 问题
文字过多过密，阅读疲劳。

### 方案

```
英雄区标题: Orbitron, 32-40px, 900 weight, letter-spacing 2px
页面标题: Orbitron, 20-24px, 700 weight
正文: Noto Sans SC, 14-16px, 行高 1.6
辅助文字: Noto Sans SC, 12px, 颜色用 text-secondary

卡片内边距: 16px (mobile) / 20px (desktop)
区块间距: 24px (mobile) / 32px (desktop)
```

## 实施顺序

1. **首页重构** — 信息层次是门面，最先改
2. **抽卡戏剧化** — 用户最常接触的核心体验
3. **动效语义化** — 建立规范，指导后续所有页面
4. **移动端适配** — 扩大用户覆盖
5. **节奏游戏偶像化** — 最难，放最后
