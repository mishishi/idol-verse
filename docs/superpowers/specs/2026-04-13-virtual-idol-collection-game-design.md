# 虚拟偶像收集养成游戏 — 设计文档

**日期**：2026/04/13
**版本**：v1.0

---

## 1. 项目概述

**项目名称**：暂定《偶像收藏集》
**项目类型**：二次元虚拟偶像收集养成 + 轻度音游 + 抽卡手游
**目标用户**：13-25岁青少年/年轻成人，尤其Z世代二次元爱好者
**核心卖点**：精美角色立绘与Live2D动画 + 碎片化轻度音游 + 成熟抽卡体系

---

## 2. 核心玩法

### 2.1 角色收集与养成

- **角色来源**：通过抽卡和碎片合成获取虚拟偶像角色
- **角色属性**：
  - 稀有度：N / R / SR / SSR / UR
  - 角色立绘（Live2D动画）
  - 独立语音和台词
  - 剧情碎片（收集解锁角色个人剧情）
- **养成系统**：
  - 等级提升（消耗经验素材）
  - 技能升级（提升应援效果）
  - 亲密度（通过互动提升，解锁更多语音和剧情）

### 2.2 轻度音游玩法

- **偶像表演**：角色登台演出时，显示节拍音符
- **玩家操作**：点击/滑动屏幕跟上节拍
- **演出奖励**：根据准确率发放抽卡道具和角色碎片
- **体力限制**：每日演出次数有限（可付费购买额外次数）

### 2.3 应援系统

- 玩家建立个人"应援殿"，放置偶像角色
- 应援殿放置产出"应援币"（游戏内货币）
- 应援殿可装饰和升级

---

## 3. 抽卡与商业化

### 3.1 抽卡机制

- **召唤道具**：圣像石（免费/付费）、召唤券
- **保底机制**：
  - 每10抽保底一张R以上
  - 第90抽触发"小保底"（概率提升至50%）
  - 第180抽触发"大保底"（指定UP角色）
- **UP池**：限定角色周期性UP，错过需等复刻

### 3.2 付费点设计

| 付费点 | 说明 |
|--------|------|
| 抽卡 | 圣像石购买、召唤券 |
| 外观 | 服装、背景、特效、应援殿装饰 |
| 体力购买 | 加速日常任务节奏 |
| 月卡 | 每日领取圣像石+体力 |
| 通行证 | 赛季任务奖励升级 |

---

## 4. 社交与成就系统

### 4.1 社交功能

- **体力互赠**：玩家每日可向好友赠送3次体力
- **应援留言板**：为偶像角色留言，应援板展示
- **偶像应援榜**：玩家为喜欢的偶像应援，排名竞争

### 4.2 成就/收集系统

- **图鉴**：收集角色解锁成就条
- **里程碑奖励**：累计抽卡次数、登录天数、演出次数等
- **亲密度成就**：与角色互动达到特定阶段解锁奖励

---

## 5. 技术架构

### 5.1 前端

- **框架**：React + TypeScript
- **动画**：Live2D（角色立绘）、Lottie（特效动画）
- **音游节拍**：Web Audio API
- **打包**：Electron（桌面端）或 响应式网页（移动端优先）

### 5.2 后端

- **框架**：Node.js + Express
- **数据库**：SQLite（开发阶段）/ PostgreSQL（生产阶段）
- **鉴权**：JWT Token

### 5.3 资源获取

- **角色立绘**：外包约稿 或 购买素材
- **语音**：TTS合成（初期）或 外包录制（后期）
- **音乐**：购买版权或外包制作

---

## 6. 开发阶段规划（建议）

### Phase 1：核心框架
- 项目初始化（React + Electron）
- 抽卡系统基础实现
- 用户注册登录

### Phase 2：角色与养成
- 角色数据结构和立绘展示
- 养成系统（等级/技能/亲密度）
- 应援殿基础功能

### Phase 3：音游玩法
- 节拍系统实现
- 演出界面和奖励结算
- 音游与抽卡联动

### Phase 4：社交与扩展
- 好友系统、体力互赠
- 应援排行榜
- 外观商城、月卡通行证

---

## 7. 成功指标

- **DAU（日活跃用户）**：目标 1000+
- **次留**：目标 40%+
- **付费率**：目标 3-5%
- **平均每付费用户收入（ARPPU）**：目标 50元+

---

## 8. 风险与应对

| 风险 | 应对 |
|------|------|
| 美术资源不足 | 初期使用免费素材/AI辅助，逐步替换 |
| 音游手感难调 | Phase 3重点测试，可简化节拍难度 |
| 抽卡合规性 | 确保概率公开、保底机制合规 |
| 独立开发工期 | Phase 1优先核心体验，非核心后期迭代 |
